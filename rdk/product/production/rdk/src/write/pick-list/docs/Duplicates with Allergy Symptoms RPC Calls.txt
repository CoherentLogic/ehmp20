When it comes to anything that involves pagination in CPRS (ex. searching for a patient)
if there are more than 44 results, you typically get back exactly 44 records (though you can get more - see below).
At that point, we call the exact same RPC call again passing in the value of the name from the last record
(typically the 44th record). This would continue until we receive less than 44 records.

NOTE:
Typically, you will receive exactly 44 records back; however, it is possible to get more than 44 records back when making
the same RPC call with some searchString's but not others.
Suppose you had 113 records all with the same name in VistA and you requested a record that was the one prior to this,
you'd probably expect to get 44 records back (with 43 of them having identical names) but you would in fact receive
114 records (the one before plus the 113 with the same name).
Suppose record 44 was the start of those 113 records, you'd get 157 records back.


An interesting result was discovered after implementing this.  We are receiving duplicates
from the RPC call (and those duplicates are also shown in CPRS).

An example of a duplicate:
--------------------------
Calling 'ORWDAL32 SYMPTOMS' with 'ALT INCREASED' (last result from 1st call) returns
480^ALT INCREASED	<INCREASED SERUM ALT (SGPT)>^INCREASED SERUM ALT (SGPT)
This is the exact same record as the last one from the 1st call - a duplicate.

An example where there is no duplicate:
---------------------------------------
Calling 'ORWDAL32 SYMPTOMS' with 'DISINHIBITION' (last result from 5th call) returns
197^DISORIENTATED	<DISORIENTED>^DISORIENTED
This record is different than the last record from the previous call.
